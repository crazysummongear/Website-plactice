# Phase 3: 銀行・クレカ自動連携機能 — コスト見積もり

**作成日**: 2026年5月6日  
**目的**: 将来的な自動連携機能実装のためのコスト試算と技術選定の参考資料

---

## 1. 概要

マネーフォワードのような**銀行口座・クレジットカード自動連携機能**を実装する場合のコスト見積もりと実装方法をまとめる。

### 目標機能
- 銀行口座の残高・取引履歴の自動取得
- クレジットカードの利用明細の自動取得
- 取引データの自動分類・カテゴリ付け
- 定期的な同期（1時間〜24時間ごと）

### 2.2 スクレイピング（自前実装）

**⚠️ 非推奨**: 以下の理由により推奨しない

#### リスク
1. **利用規約違反**: 多くの銀行・クレカサイトは自動アクセスを禁止
2. **セキュリティリスク**: 
   - ユーザーの銀行認証情報（ID/パスワード）を保存する必要がある
   - 情報漏洩時の責任問題
3. **メンテナンスコスト**: 
   - 銀行サイトのUI変更に対応する必要がある
   - 各金融機関ごとに個別実装が必要
4. **法的リスク**: 不正アクセス禁止法に抵触する可能性

#### 実装コスト（参考）
- 開発工数: 1金融機関あたり20〜40時間
- メンテナンス: 月5〜10時間（サイト変更対応）
- AWS Lambda実行コスト: $0.5〜$2/月

### 3.2 データフロー

#### 初回連携フロー
```
1. ユーザーが「口座連携」ボタンをクリック
   ↓
2. フロントエンドが Moneytree OAuth URLにリダイレクト
   ↓
3. ユーザーが銀行サイトでログイン・認証
   ↓
4. Moneytree が認証コードを返す
   ↓
5. Lambda が認証コードをアクセストークンに交換
   ↓
6. アクセストークンを DynamoDB に保存（暗号化）
   ↓
7. 初回同期を実行
```

#### 定期同期フロー
```
1. EventBridge が1時間ごとに sync-accounts Lambda を実行
   ↓
2. Lambda が DynamoDB から連携口座一覧を取得
   ↓
3. 各口座について Moneytree API を呼び出し
   ↓
4. 新しい取引データを取得
   ↓
5. 自動カテゴリ分類（AI/ルールベース）
   ↓
6. DynamoDB に保存
   ↓
7. ユーザーに通知（オプション）
```

## 5. スケールメリット分析

### 5.1 ユーザー数別コスト

個人利用（1ユーザー）と複数ユーザーでのコスト比較

| ユーザー数 | AWS コスト | Moneytree コスト | 合計 | 1ユーザーあたり |
|-----------|-----------|-----------------|------|----------------|
| **1人** | $1.28 | $35 | **$36.28** | **$36.28** |
| **10人** | $3.50 | $50 | **$53.50** | **$5.35** |
| **100人** | $15.00 | $100 | **$115.00** | **$1.15** |
| **1,000人** | $80.00 | $300 | **$380.00** | **$0.38** |

### 5.2 考察

- **個人利用**: 1ユーザーあたり$36/月と高コスト
- **10ユーザー**: 1ユーザーあたり$5.35/月（約85%削減）
- **100ユーザー**: 1ユーザーあたり$1.15/月（約97%削減）
- **1,000ユーザー**: 1ユーザーあたり$0.38/月（約99%削減）

**結論**: 自動連携機能は**スケールメリットが非常に大きい**。個人利用では費用対効果が低いが、複数ユーザーで共有すれば大幅にコスト削減可能。

## 6. 代替案: Phase 2（半自動化）

Phase 3の完全自動化が高コストな場合の代替案

### 6.1 ブラウザ拡張機能 + CSV自動ダウンロード

#### 実装方法
```

   ↓
2. 銀行・クレカサイトにアクセス
   ↓
3. CSVダウンロードボタンを自動クリック
   ↓
4. ダウンロードしたCSVを自動でS3にアップロード
   ↓
5. Lambda がCSVをパース・DynamoDBに保存
```

#### コスト
- **開発コスト**: 20〜40時間
- **AWS コスト**: $0.5〜$1/月
- **外部サービス**: $0（不要）
- **合計**: **$0.5〜$1/月**

#### メリット
- 低コスト
- 銀行の利用規約に準拠
- セキュリティリスクが低い

#### デメリット
- 完全自動ではない（ユーザーが定期的にブラウザ拡張を実行する必要がある）
- 各金融機関ごとに個別実装が必要

---

### 6.2 定期的なCSV手動アップロード

#### 実装方法
```
1. ユーザーが月1回、銀行・クレカサイトからCSVをダウンロード
   ↓
2. アプリのCSVインポート画面でアップロード
   ↓
3. Lambda がCSVをパース・DynamoDBに保存
```

#### コスト
- **開発コスト**: 0時間（既存機能）
- **AWS コスト**: $0.13〜$0.5/月
- **外部サービス**: $0（不要）
- **合計**: **$0.13〜$0.5/月**

#### メリット
- 最低コスト
- 既存機能で実現可能
- セキュリティリスクなし

#### デメリット
- 手動作業が必要
- リアルタイム性がない

### 8.3 Moneytree Link の詳細

- **提供元**: Moneytree株式会社（2012年設立）
- **サービス開始**: 2015年
- **対応金融機関**: 2,369種類（2026年5月時点）
- **企業パートナー**: 60社以上（メガバンク、地方銀行、会計ソフト等）

#### 提供形態

1. **Moneytree Link API**
   - 法人向けAPI
   - 金融機関データの取得・集約
   - OAuth 2.0認証

2. **LINK Kit**
   - Web/モバイル向けPFM（個人財務管理）ソリューション
   - 埋め込み型UI

#### 契約条件

- **対象**: 法人のみ（個人開発者は契約不可）

#### GitHubで公開されているもの

Moneytree が GitHub で公開しているのは**クライアント側SDKのみ**：

| リポジトリ | 内容 | ライセンス |
|-----------|------|-----------|
| `mt-link-javascript-sdk` | JavaScript SDK | MIT |
| `moneytree-link-ios-sdk` | iOS SDK | MIT |
| Android SDK | Android SDK | MIT |

**重要**: これらは「Moneytree Link APIを呼び出すためのライブラリ」であり、「金融機関連携システム本体」ではありません。

#### 誤解されやすいポイント

❌ **誤解**: GitHubのSDKをデプロイすれば金融機関連携ができる  
✅ **正解**: SDKはクライアント側のライブラリ。サーバー側（金融機関連携の核心部分）は非公開。

```
【公開されているもの】
あなたのアプリ
  ↓
mt-link-javascript-sdk (GitHub公開、MITライセンス)
  ↓
Moneytree Link API (Moneytreeのサーバー、要契約)
  ↓
銀行・クレカ

【公開されていないもの】
Moneytree Link API (Moneytreeのサーバー)
├─ 金融機関連携ロジック
├─ データパース・整形

```

### 8.5 スクレイピング実装のリスク

多くの個人開発者が検討する「スクレイピング」には、以下のリスクがあります：

#### 法的リスク

1. **利用規約違反**
   - 多くの銀行・クレカサイトは自動アクセスを禁止
   - 例: 「ロボット等による自動アクセスを禁止します」

2. **不正アクセス禁止法**
   - 利用規約違反の自動アクセスは、不正アクセス禁止法に抵触する可能性
   - 罰則: 3年以下の懲役または100万円以下の罰金

#### セキュリティリスク

1. **認証情報の保存**
   - ユーザーの銀行ID/パスワードを保存する必要がある
   - 情報漏洩時の責任が重大

2. **中間者攻撃**
   - 通信の傍受リスク

#### 技術的リスク

1. **メンテナンスコスト**
   - 銀行サイトのUI変更に対応し続ける必要がある
   - 各金融機関ごとに個別実装が必要

2. **エラーハンドリング**
   - ログイン失敗、タイムアウト、セッション切れ等

#### 実装コスト

```
1金融機関あたり:
- 開発工数: 20〜40時間
- メンテナンス: 5〜10時間/月
- エラー対応: 随時

10金融機関の場合:
- 初期開発: 200〜400時間
- 月間メンテナンス: 50〜100時間
```

**結論**: 個人開発でのスクレイピング実装は**非推奨**

### 8.6 推奨アプローチ（個人開発）

個人開発・学習目的の場合、以下の段階的アプローチを推奨します：

#### Phase 1: MVP（推奨）
- **機能**: 手動入力 + CSV手動インポート
- **工数**: 0時間（既存機能）
- **コスト**: $0.13〜$0.5/月
- **リスク**: なし
- **メリット**: 
  - 低コスト
  - 法的リスクなし
  - すぐに使える

#### Phase 2: 半自動化（推奨）
- **機能**: ブラウザ拡張 + CSV自動ダウンロード
- **工数**: 20〜40時間
- **コスト**: $0.5〜$1/月
- **リスク**: 低
- **メリット**:
  - 低コスト
  - 利用規約に準拠
  - ユーザー体験の向上

#### Phase 3: 完全自動化（将来・条件付き）
- **機能**: Moneytree Link API連携
- **工数**: 40〜80時間
- **コスト**: $35〜$70/月
- **リスク**: なし
- **条件**:
  - 法人化
  - または、複数ユーザーで共有（10人以上）
  - または、個人で月額$35を許容できる場合
