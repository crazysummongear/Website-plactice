# kakei — 要件定義書

**プロジェクト名**: kakei（家計管理アプリ）  
**目標**: 35歳での FIRE（経済的自由）達成を支援する個人向け資産管理アプリ  
**作成日**: 2026年5月4日

---

## 1. ビジネス要件

### 1.1 プロジェクト概要

| 項目 | 内容 |
|------|------|
| **アプリ名** | kakei（家計簿） |
| **コンセプト** | スマホ最適化・低コスト運用の個人向け家計簿アプリ |
| **ターゲット** | 個人ユーザー（単一アカウント〜家族共有） |
| **運用コスト目標** | 月額 $5 以内（個人学習・個人利用） |
| **開発スタイル** | AI 駆動開発（Kiro + Terraform IaC） |

### 1.2 ビジネス目標

- **短期（3ヶ月）**: MVP（最小限の機能）をリリース
  - ユーザー認証（Cognito）
  - 収支入力・表示
  - CSV インポート
  
- **中期（6ヶ月）**: 機能拡張
  - 予算管理・アラート
  - カテゴリ別分析
  - 家族共有機能

- **長期（12ヶ月）**: 本格運用
  - モバイルアプリ化
  - AI による支出予測
  - 資産管理機能（投資ポートフォリオ）

---

## 2. 機能要件

### 2.1 ユーザー認証

**要件 ID**: REQ-AUTH-001

| 項目 | 内容 |
|------|------|
| **機能** | メール + パスワード認証 |
| **実装** | Amazon Cognito User Pool |
| **フロー** | サインアップ → メール検証 → ログイン |
| **セッション管理** | JWT トークン（ID Token + Access Token） |
| **パスワードリセット** | メール経由でのリセットリンク |

**受け入れ基準**:
- [ ] ユーザーがメールアドレスとパスワードでサインアップできる
- [ ] メール検証コードが送信される
- [ ] 検証後、ログインできる
- [ ] JWT トークンが API リクエストに含まれる
- [ ] パスワードリセット機能が動作する

---

### 2.2 収支入力・管理

**要件 ID**: REQ-TX-001

| 項目 | 内容 |
|------|------|
| **機能** | 日々の収支を記録・管理 |
| **入力項目** | 日付、カテゴリ、金額、収入/支出区分、メモ |
| **操作** | 新規作成、編集、削除、一覧表示 |
| **フィルタ** | 期間、カテゴリ、収入/支出で絞り込み |
| **ストレージ** | DynamoDB（Single Table Design） |

**受け入れ基準**:
- [ ] ユーザーが収支を入力できる
- [ ] 入力した収支が一覧に表示される
- [ ] 収支を編集・削除できる
- [ ] 期間・カテゴリでフィルタできる
- [ ] 月別・カテゴリ別の合計が表示される

---

### 2.3 CSV インポート

**要件 ID**: REQ-CSV-001

| 項目 | 内容 |
|------|------|
| **機能** | 銀行・クレジットカード明細を一括取り込み |
| **フロー** | ファイル選択 → S3 アップロード → 自動パース → DynamoDB 保存 |
| **重複チェック** | 同一日付・金額の取引は重複として検出 |
| **カラムマッピング** | ユーザーが CSV カラムを指定 |
| **ストレージ** | S3（一時保存、7日後自動削除） |

**受け入れ基準**:
- [ ] ユーザーが CSV ファイルをアップロードできる
- [ ] CSV がパースされ、DynamoDB に保存される
- [ ] 重複チェックが機能する
- [ ] アップロード進捗が表示される

---

### 2.4 ダッシュボード・分析

**要件 ID**: REQ-DASHBOARD-001

| 項目 | 内容 |
|------|------|
| **機能** | 月別・カテゴリ別の収支サマリー表示 |
| **表示内容** | 今月の収入、支出、残高、カテゴリ別内訳 |
| **グラフ** | 円グラフ（カテゴリ別）、折れ線グラフ（月別推移） |
| **期間選択** | 月単位で切り替え可能 |

**受け入れ基準**:
- [ ] ダッシュボードに月別サマリーが表示される
- [ ] カテゴリ別の内訳が表示される
- [ ] 月を切り替えると表示が更新される

## 3. 非機能要件

### 3.1 パフォーマンス

| 項目 | 要件 |
|------|------|
| **API レスポンス時間** | < 500ms（p95） |
| **フロントエンド初期ロード** | < 2秒（3G） |
| **DynamoDB クエリ** | < 100ms（p95） |

### 3.2 スケーラビリティ

| 項目 | 要件 |
|------|------|
| **同時ユーザー数** | 初期: 1、将来: 10 |
| **月間 API 呼び出し** | 初期: 300、将来: 10,000 |
| **DynamoDB スループット** | オンデマンド（自動スケーリング） |

### 3.3 セキュリティ

| 項目 | 要件 |
|------|------|
| **認証** | Cognito User Pool + JWT |
| **認可** | API Gateway Cognito Authorizer |
| **通信** | HTTPS 強制 |
| **データ暗号化** | DynamoDB 暗号化（AWS 管理キー） |
| **IAM** | 最小権限の原則 |

### 3.4 可用性・信頼性

| 項目 | 要件 |
|------|------|
| **SLA** | 99.9%（月間ダウンタイム < 43分） |
| **バックアップ** | DynamoDB PITR（Point-in-Time Recovery） |
| **ディザスタリカバリ** | 別リージョンへの自動フェイルオーバー（将来） |

### 3.5 コスト

| 項目 | 要件 |
|------|------|
| **月額運用コスト** | $0.5 以内（個人学習・個人利用） |
| **コスト可視化** | AWS Budgets + タグ付け |
| **コスト最適化** | DynamoDB オンデマンド、S3 ライフサイクル、Lambda 最小メモリ、CloudFront PriceClass_100 |

**注意**: 個人利用（月間300リクエスト程度）の場合、実際のコストは$0.16〜$0.5/月程度に収まる見込み。

## 4. 技術要件

### 4.1 インフラストラクチャ

| コンポーネント | 技術 | 理由 |
|---------------|------|------|
| **IaC** | Terraform | 再現性・バージョン管理 |
| **クラウド** | AWS | 無料枠・豊富なサービス |
| **リージョン** | ap-northeast-1（東京） | レイテンシ最小化 |

### 4.2 バックエンド

| コンポーネント | 技術 | 理由 |
|---------------|------|------|
| **ランタイム** | Node.js 20.x | Lambda 対応・高速 |
| **言語** | TypeScript | 型安全性 |
| **フレームワーク** | AWS Lambda | サーバーレス・低コスト |
| **DB** | DynamoDB | NoSQL・スケーラブル |

### 4.3 フロントエンド

| コンポーネント | 技術 | 理由 |
|---------------|------|------|
| **フレームワーク** | React 18 | コンポーネント指向 |
| **言語** | TypeScript | 型安全性 |
| **スタイリング** | Tailwind CSS | ユーティリティファースト |
| **ビルドツール** | Vite | 高速ビルド |
| **ホスティング** | S3 + CloudFront | 静的ホスティング・CDN |

### 4.4 認証・認可

| コンポーネント | 技術 | 理由 |
|---------------|------|------|
| **ユーザー管理** | Cognito User Pool | マネージドサービス |
| **API 認可** | API Gateway Cognito Authorizer | JWT 検証 |
| **SSO** | AWS Identity Center | 個人開発向け |

## 5. 制約・前提条件

### 5.1 制約

- **個人開発**: 1 人で開発・運用

### 5.2 前提条件

- AWS アカウント
- AWS CLI インストール済み
- Node.js 20.x インストール済み
- Git / GitHub アカウント
